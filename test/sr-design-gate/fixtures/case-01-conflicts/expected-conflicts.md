# 预期语义冲突

1. `CONFLICT-001`：§4 超时 3 秒，§9 和 §10 为 5 秒。
2. `CONFLICT-002`：§5.7 没有 `CANCELING`，§10 使用该状态。
3. `CONFLICT-003`：§7.1 与 §7.2 的 AR-001/AR-002 依赖方向相反。
4. `CONFLICT-004`：§5.3 是同步调用，§7.3 是异步消息。
5. `CONFLICT-005`：§6 的 P99 为 200ms，§10 为 500ms。
6. `CONFLICT-006`：§5.8 的 `requestId` 为 integer，§4 为 string。
7. `CONFLICT-007`：§5.1 和 §5.9 失败后重试，§5.4 和 §10 直接终止。
8. `CONFLICT-008`：AR 概览包含 AR-003，但需求追溯明确没有需求映射。
9. `CONFLICT-009`：通知发送定义为范围外，却由 AR-002 承接。
10. `CONFLICT-010`：§5.2 存在软件架构禁止的反向依赖。
