# 模块测试用例设计模板

使用本模板创建或更新《{AR编号}-{需求短名}-{模块名}模块测试用例设计.md》。本文件是独立测试设计成果物，基于正式《模块详细设计说明书.md》中的
TOBE 设计和可测试性输入生成；不得替 TOBE 补设计，不得拆分 AICoding 任务。

正式测试设计只输出验证结论，不记录目标抽取、风险分级、候选用例、用例合并、淘汰理由等推导过程。Gate
会基于本文件的最小充分测试集、覆盖矩阵和缺口反查测试设计质量。

## 排版约定

第 2 节「最小充分测试用例集」采用**列表式分层**呈现，不用宽表平铺：

- `### 2.x <测试层级>`：顶层按测试层级分组（单元 / 模块内功能集成 / 契约 / 兼容回归 / 数据迁移 / 配置开关 / 观测性 / 风险专项 / 人工验证等）。**只建立本次设计能支撑的层级**，不强求把每个层级都列出来；能做模块内功能集成甚至端到端的，就建对应层级分组，没有就省略。
- `#### 2.x.y <功能点 / 场景标题>`：层级下按功能点或场景分组，标题自定。
- `##### TCn <用例简述>`：场景下每个用例一个条目，用列表平铺字段。

覆盖矩阵（第 3 节）和缺口回流（第 4 节）仍用表格，便于 Gate 反查。
